# Requirements Document

## Introduction

AI Personal Study Planner is a next-generation, GenAI-powered web application that acts as a smart AI study mentor. The system understands student needs, generates personalized study plans, continuously adapts to student performance, and provides an intelligent tutoring experience. It combines study planning, AI summarization, quiz generation, weak topic detection, dynamic rescheduling, chatbot tutoring, productivity insights, stress management, and smart revision planning into a single cohesive platform.

## Glossary

- **Study Planner**: The AI Personal Study Planner web application
- **Student**: An authenticated user of the Study Planner
- **Study Plan**: A personalized daily/weekly schedule of study tasks generated by AI
- **AI Model**: A Hugging Face-hosted language model used for AI features
- **Flan-T5**: The `google/flan-t5-large` Hugging Face model used for study plan generation
- **BART**: The `facebook/bart-large-cnn` Hugging Face model used for summarization
- **Mistral**: The `mistralai/Mistral-7B-Instruct` Hugging Face model used for quiz generation and chatbot
- **Weak Topic**: A subject or topic where the Student has demonstrated low performance
- **Forgetting Curve**: The psychological model describing how memory fades over time without revision
- **Revision Slot**: A scheduled time block dedicated to reviewing previously studied material
- **JWT**: JSON Web Token used for stateless authentication
- **In-Memory Storage**: Server-side data storage using JavaScript objects/arrays without a persistent database
- **Glassmorphism**: A UI design style using frosted glass effects with transparency and blur
- **MCQ**: Multiple Choice Question
- **Productivity Insight**: An AI-generated observation about a Student's study habits and patterns
- **Dynamic Rescheduler**: The AI component that regenerates a Study Plan when tasks are missed

## Requirements

### Requirement 1

**User Story:** As a student, I want to register and log in securely, so that my study data and plans are private and personalized to me.

#### Acceptance Criteria

1. WHEN a user submits a registration form with a username and password, THE Study Planner SHALL create a new account and store credentials in memory with a hashed password.
2. WHEN a user submits valid login credentials, THE Study Planner SHALL return a signed JWT token with a 24-hour expiry.
3. WHEN a request is made to a protected API endpoint without a valid JWT, THE Study Planner SHALL return a 401 Unauthorized response.
4. IF a user submits a registration form with a username that already exists, THEN THE Study Planner SHALL return an error message indicating the username is taken.
5. IF a user submits login credentials with an incorrect password, THEN THE Study Planner SHALL return a 401 error without revealing which field is incorrect.

---

### Requirement 2

**User Story:** As a student, I want to generate a personalized study plan, so that I can organize my preparation efficiently based on my subjects, exam dates, and available hours.

#### Acceptance Criteria

1. WHEN a student provides subjects, exam dates, daily study hours, and weak topics, THE Study Planner SHALL call the Flan-T5 model to generate a structured daily and weekly study plan.
2. WHEN the AI model returns a study plan, THE Study Planner SHALL parse and store the plan in memory associated with the authenticated student.
3. WHEN a student views the dashboard, THE Study Planner SHALL display the current active study plan with daily tasks and progress indicators.
4. IF the AI model call fails or times out, THEN THE Study Planner SHALL return a descriptive error message and preserve the student's previous plan.
5. WHILE a study plan is being generated, THE Study Planner SHALL display a loading skeleton in the UI.

---

### Requirement 3

**User Story:** As a student, I want to summarize my notes using AI, so that I can quickly review key concepts without reading lengthy material.

#### Acceptance Criteria

1. WHEN a student submits text content to the summarizer, THE Study Planner SHALL call the BART model and return bullet-point summaries highlighting key concepts.
2. WHEN the BART model returns a summary, THE Study Planner SHALL display the output formatted as a bulleted list within 30 seconds of submission.
3. IF the submitted text is fewer than 50 characters, THEN THE Study Planner SHALL reject the request and display a validation error without calling the AI model.
4. IF the AI summarization call fails, THEN THE Study Planner SHALL display an error toast notification and allow the student to retry.

---

### Requirement 4

**User Story:** As a student, I want to generate AI-powered quizzes on any topic, so that I can test my knowledge and identify gaps.

#### Acceptance Criteria

1. WHEN a student specifies a topic and quiz type (MCQ, one-word, or match-the-following), THE Study Planner SHALL call the Mistral model to generate a quiz with a minimum of 5 questions.
2. WHEN the quiz is generated, THE Study Planner SHALL display questions with interactive answer inputs and a submit button.
3. WHEN a student submits quiz answers, THE Study Planner SHALL evaluate responses and display a score with correct answers highlighted.
4. WHEN a student completes a quiz, THE Study Planner SHALL store the quiz result (topic, score, timestamp) in memory for weak topic analysis.
5. IF the quiz generation call fails, THEN THE Study Planner SHALL display an error message and preserve any previously generated quiz.

---

### Requirement 5

**User Story:** As a student, I want the system to detect my weak topics automatically, so that I can focus my study efforts where they matter most.

#### Acceptance Criteria

1. WHEN a student requests weak topic analysis, THE Study Planner SHALL send quiz performance data and self-reported difficulty ratings to the Mistral model and return identified weak areas with improvement suggestions.
2. WHEN weak topics are identified, THE Study Planner SHALL display them on the dashboard with color-coded indicators (red for critical, yellow for moderate, green for strong).
3. WHEN weak topics are detected, THE Study Planner SHALL automatically suggest a focused improvement plan as part of the AI Suggestions panel.
4. IF no quiz data exists for a student, THEN THE Study Planner SHALL prompt the student to complete at least one quiz before running weak topic analysis.

---

### Requirement 6

**User Story:** As a student, I want the AI to reschedule my study plan when I miss tasks, so that I can stay on track without feeling overwhelmed.

#### Acceptance Criteria

1. WHEN a student marks tasks as missed or skipped, THE Study Planner SHALL call the Flan-T5 model with the missed tasks and remaining schedule to generate a revised plan.
2. WHEN the rescheduled plan is generated, THE Study Planner SHALL replace the current plan in memory and display the updated schedule to the student.
3. WHEN rescheduling, THE Study Planner SHALL distribute missed tasks across remaining days without exceeding the student's configured daily study hours.
4. IF all remaining days are insufficient to cover missed tasks, THEN THE Study Planner SHALL notify the student and suggest extending the study period or reducing scope.

---

### Requirement 7

**User Story:** As a student, I want AI-powered study recommendations, so that I know what to study next and when to revise based on my habits and weak areas.

#### Acceptance Criteria

1. WHEN a student views the dashboard, THE Study Planner SHALL display an AI Suggestions panel with at least 3 personalized recommendations based on weak topics and study history.
2. WHEN generating recommendations, THE Study Planner SHALL factor in the student's weak topics, last study session, and upcoming exam dates.
3. WHEN a student dismisses a recommendation, THE Study Planner SHALL remove it from the panel and not show the same recommendation again in the same session.

---

### Requirement 8

**User Story:** As a student, I want to chat with an AI tutor, so that I can get instant explanations and concept clarifications at any time.

#### Acceptance Criteria

1. WHEN a student sends a message in the chatbot interface, THE Study Planner SHALL call the Mistral model and display the AI response within the chat thread.
2. WHEN the chatbot responds, THE Study Planner SHALL maintain the full conversation history within the session and send it as context with each new message.
3. WHEN the chatbot is available, THE Study Planner SHALL display a floating chat button on all pages for quick access.
4. IF the Mistral model call fails, THEN THE Study Planner SHALL display an inline error message in the chat thread and allow the student to resend the message.

---

### Requirement 9

**User Story:** As a student, I want AI productivity insights about my study habits, so that I can understand my patterns and improve my efficiency.

#### Acceptance Criteria

1. WHEN a student views the Insights page, THE Study Planner SHALL display AI-generated observations about study hours, completion rates, and behavioral patterns.
2. WHEN generating insights, THE Study Planner SHALL analyze stored session data and produce at least 2 specific, actionable observations (e.g., peak productivity times, procrastination patterns).
3. WHEN insights are displayed, THE Study Planner SHALL present them as styled insight cards with icons and concise descriptions.

---

### Requirement 10

**User Story:** As a student, I want the AI to suggest breaks and motivational support, so that I can manage stress and maintain consistent study habits.

#### Acceptance Criteria

1. WHEN a student has logged more than 2 consecutive hours of study activity, THE Study Planner SHALL display a break suggestion with a recommended duration.
2. WHEN a student opens the dashboard, THE Study Planner SHALL display a motivational quote relevant to their current progress.
3. WHEN a student's task completion rate drops below 50% in a week, THE Study Planner SHALL display an encouraging message and suggest a lighter schedule.

---

### Requirement 11

**User Story:** As a student, I want a smart revision planner based on the forgetting curve, so that I automatically get revision slots scheduled before I forget material.

#### Acceptance Criteria

1. WHEN a student marks a topic as studied, THE Study Planner SHALL automatically schedule revision slots at intervals based on the forgetting curve (1 day, 3 days, 7 days, 14 days).
2. WHEN revision slots are due, THE Study Planner SHALL highlight them on the calendar view and include them in the daily plan.
3. WHEN a student completes a revision slot, THE Study Planner SHALL reschedule the next revision at a longer interval.

---

### Requirement 12

**User Story:** As a student, I want a polished, responsive UI with dark/light mode, so that I have a comfortable and visually appealing study experience.

#### Acceptance Criteria

1. THE Study Planner SHALL default to dark mode with a purple-to-indigo gradient theme and glassmorphism card components.
2. WHEN a student toggles the theme switch, THE Study Planner SHALL switch between dark and light mode and persist the preference in local storage.
3. WHEN data is loading from an AI model, THE Study Planner SHALL display animated skeleton loaders in place of content.
4. WHEN an action succeeds or fails, THE Study Planner SHALL display a toast notification with an appropriate message and auto-dismiss after 4 seconds.
5. THE Study Planner SHALL render correctly on screen widths from 320px to 2560px without horizontal scrolling.
6. WHEN a student navigates to any page, THE Study Planner SHALL display a calendar and timeline view of the current study plan.
